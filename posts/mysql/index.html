数据库基础

行

表中的数据是按行存储的，所保存的每个记录存储在自己的行内

例如，顾客表可以每行存储一个顾客。表中的行数为记录的总数。

列

表中的一个字段。所有表都是由一个或多个列组成的

理解列的最好办法是将数据库表想象为一个网格。网格中每一列存

储着一条特定的信息

例如，在顾客表中，一个列存储着顾客编号，另一个列存储着顾客名，而地址、城市、州以及邮政编码全都存储在各自的列中。

数据类型

数据类型限制可存储在列中的数据种类（例如，防止在数值字段中

录入字符值）。数据类型还帮助正确地排序数据，并在优化磁盘使用方面

DECIMAL类型

DECIMAL[M,D] [UNSIGNED]

可以支持更加精准的小数位，M是小数位数的总数，D是小数点后面的位数

如果D=0，则没有小数点或分数部分，M最大=65，D最大=30，默认情况下，M=10，D=0

如果希望小数的精度高，double基本够用，非常高建议使用decimal

字符串类型

char类型最大255字符

varchar可变长度字符串最大0-65535字节

(如果使用utf8编码最大218444字符，因为需要1-3个字节用于记录大小，所以是(65535-3)/3=21844)

使用gbk编码则是（65535-3)/2=32766

字符串使用细节

char(4)这个4表示字符数（最大255），不是字节数，无论是中文还是字母都只能放四个

char（4）是定长，即使只插入‘aa'也会占用4个字符的空间，造成空间浪费

varchar（4）是边长，如果插入’aa‘，实际占用空间并不是4个字符，而是按照实际占用空间分配。（varchar本身需要占用1-3个字节来存放内容长度）

什么时候用char，varchar（查询速度cahr>varchar）

  1. 如果数据是定长，推荐使用char，比如md5的密码，邮编，手机号，身份证号
  2. 如果一个字段长度不确定，推荐使用varchar，比如留言，文章

varchar不够用，可用text，mediumtext，longtext

insert

INSERT INTO `goods`(id,goods_name,price)	(列名)
	VALUES(1,'huawei',3000); （数据）

detail

  1. 插入数据应与字段的数据类型相同（类型不匹配是，底层会尝试类型转换，不成功就报错）
  2. 数据长度应在列的规定范围内
  3. 在values中列出的数据位置必须与被加入的列的排列位置相对应
  4. 字符和日期类型数据应包含在单引号'‘中
  5. 列可以插入空值（前提是该字段允许为空，NOT NULLz则不允许为空）
  6. INSERT INTO 'goods'(列名，...) VALUES(,,,),(,,)可以实现一次插入多条数据
  7. 如果是给表中所有字段添加数据，可以不写前面的字段名称，（会默认匹配，但需要数量匹配）
  8. 如果某个列，没有指定not null，那么当添加数据时，则默认值给null。设置默认值语句：NOT NULL DEFAULT 值

mysql约束

主键

  • primary key 不能重复且不为null

  • 一张表只能有一个主键，但可以有复合主键

  • 主键指定方式

    • 直接在字段后指定 字段名 primary key
    • 在表定义最后写 primary key（列明）

  • 使用desc 表名可看到primary key的情况

Not null

定义了not null那么插入数据时必须有数据不能为空

Unique

定了unique表示该列值不能重复

外键

用于定义主表和从表之间的关系，外键约束要定义在从表上，主表必须有主键约束或者unique约束

当定义外键约束后，要求外键列数据必须在主表的主键列存在或者为null

//主表
CREATE TABLE my_class(
	id INT PRIMARY KEY,  主表中设置主键约束
	`name` VARCHAR(32) NOT NULL DEFAULT '');
	
//从表	
CREATE TABLE my_stu(
	id INT PRIMARY KEY
	`name` VARCHAR(32) NOT NULL DEFAULT '',
	class_id INT,
	设置从表的class_id以主表的id作为参考，主表id中不存在的数据，从表中无法加入
	FOREIGN KEY (class_id) REFERENCES my_class(id))
	
INSERT INTO my_class VALUES(100,'c'),(200,'test');
SELECT * FROM my_class

INSERT INTO my_stu VALUES(1,'felix',100),(2,'tom',200)
SELECT *FROM my_stu

主表id中不存在的数据，从表中无法加入
INSERT INTO my_stu VALUES(3,'jack',300)

detail

  • 外键指向的表的字段，要求
  • 是primary key或者是unique
  • 表的类型是innodb，其他的不支持外键
  • 外键字段的类型要和主键字段的类型一致（长度可以不同）
  • 外键字段的值，必须在主键字段中出现过，或者为null（前提是允许为null）
  • 主外键关系建立后，数据不能随意删除，要先删外键数据再删主表数据
